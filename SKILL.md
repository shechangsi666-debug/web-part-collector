# 网页信息采集器

适用场景：用户提供供应商官网URL，需要采集四轮一带（底盘件）的产品信息。

## 采集流程

1. **识别网站结构** ── 分析URL所在网站的产品目录结构
2. **进入产品页面** ── 导航到Products/Parts/Catalog 页面
3. **筛选底盘件** ── 识别属于四轮一带来的产品（用 is_crawler_part() 过滤）
4. **提取字段** ── 从产品页提取件号、名称、描述、图片等
5. **下载图片** ── 仅下载产品图/结构图/爆炸图，过滤Banner/Logo
6. **标准化命名** ── 用 standardize.standardize_name() 映射到标准名
7. **导出Excel** ── 用 export.export_to_excel() 生成12列标准表
8. **整理图片** ── 按标准名分类放入 官网图片/ 目录
9. **生成日志** ── 记录采集过程、异常、缺失字段

## 脚本使用

```bash
cd scripts/
python collect.py <官网URL> [--output-dir 输出目录]
```

- collect.py 为主入口，自动加载适配器执行采集流程
- standardize.py 提供名称标准化函数（standalone可用）
- export.py 提供Excel导出+图片整理（standalone可用）
- 每个供应商网站结构不同时，在 adapters/ 下创建适配器继承 BaseAdapter

## 名称标准化

名称标准化映射表在 references/name_mapping.md。核心规则：
- 保留官网原始名称，同时生成标准英文名和标准中文名，不做替换
- 所有同义词统一到标准词：如 Track Roller / Bottom Roller / Lower Roller -> Track Roller -> 支重轮

## 输出结构

```text
输出目录/
├── Excel/          products.xlsx（12列：主机类型/品牌/型号/标准名/原始名/件号/描述/链接/图名/图路径/时间/备注）
├── 官网图片/       按标准名分类（支重轮/托链轮/引导轮/驱动齿/链条/履带板/履带总成/涨紧总成/待分类）
└── Logs/           采集日志
```

## 数据质量

- 同件号+同原始名视为同一产品，合并保留全部来源链接
- 件号缺失标记"待确认"，型号缺失标记"型号缺失"
- 图片下载失败记录原因到备注
- 无法识别的产品放入"待分类"
